JSP

九个内置对象

指令
page 指令就是配置jsp页面的.
	创建jsp自带什么属性.就用哪些属性.不需要自己配置.
	errorPage 指定出错之后,跳转的页面.
		//用这个属性去指定错误页面.需要在每个jsp页面中都指定.很麻烦.
    	//我们一般在web.xml里面统一配置.
     <error-page>
        <!--<error-code>500</error-code>--> error-page和exception-type是互斥的
        <exception-type>java.lang.NullPointerException</exception-type>
        <location>/500.jsp</location>
    </error-page>
    
	isErrorPage 表示当前页面是错误页面.  为了使用exception对象.
    
 include 引入其他的jsp文件的. 可以在任意位置引入. 引入之后就相当于把别引入的jsp内容拷贝到这里.
 
 taglib  主要就是引入标签库.

jsp的内置对象

四个与对象
pageContext     PageContext   在当前页面中共享数据.还可以获取其他八个对象.
		//还可以往其他三个域中存储数据.
    pageContext.setAttribute(String var1, Object var2, int var3) 使用这三个参数的方法
    	第三个参数是指定要将数据存储到哪个域中.分别是PageContext中定义好的是个常量
    		 PAGE_SCOPE
     		REQUEST_SCOPE 
     		SESSION_SCOPE 
     		APPLICATION_SCOPE 
     还可以用getAttribute(String var1, int var2)去指定域中获取数据.
    Object findAttribute(String var1);从小到大依次从四个域中去找这个键所对应的值. 如果找到了就结束了. 如果四个域中都没有.就返回null
    
    
     <%
        pageContext.setAttribute("msg","page");
        pageContext.setAttribute("msg","hello",PageContext.SESSION_SCOPE);

      %>

    <%
      Object msg = pageContext.getAttribute("msg");
      System.out.println(msg);
      System.out.println("==================");
      Object msg1 = session.getAttribute("msg");
      System.out.println(msg1);
      System.out.println("============");
      Object msg2 = pageContext.getAttribute("msg", PageContext.SESSION_SCOPE);
      System.out.println(msg2);
      System.out.println("==========");
      //从小到大依次从四个域中去找这个键所对应的值. 如果找到了就结束了. 如果四个域中都没有.就返回null
      Object msg3 = pageContext.findAttribute("msg");
      System.out.println(msg3);
    %>

MVC设计模式

M model  数据模型.  dao service 整个的业务逻辑层都属于model
V view  展示层
	html jsp
C Controller 调度. 将view层用户填写的数据转交给model层. 将model层处理后的数据转交给view层.
	Servlet
	
软件开发的原则:
	高内聚 : 相同的业务逻辑尽可能都放到一起.
	低耦合 : 不同的业务逻辑要区分开.

EL表达式

expression language  表达式语言
语法  ${}
作用 :
	1.运算
		java中的所有运算符el都可以用.
		empty : 就是判断获取出来的数据是否可用.
            //所有的el表达式的判断都要写到{}里面
            ${not empty 键的名称}
			${name != "张三"}
	2.获取值 (只能获取域中的值)
		pageScope
		requestScope
		sessionScope
		applicationScope
			他们四个都是Map集合.对应着四个域对象.可以从四个域中去获取对应的数据.
        1. ${键名}：表示依次从最小的域中查找是否有该键对应的值，直到找到为止。
		2. 获取对象、List集合、Map集合的值
				1. 对象：${域名称.键名.属性名}
					* 本质上会去调用对象的getter方法
				2.list
					${域名称.键名[索引]}
				3.map
					${域名称.键名.map中的key}
					${域名称.键名["map中的key"]}
					使用场景:
						map.put("bb.aa","sss");
						${map["bb.aa"]}
	
注意:
	page指令汇中的isELIgnored不同的版本默认值是不同的. 建议大家加上这个属性将值设置成false.
        
        
 隐式对象:  11个
	PageContext 可以获取其他jsp中的8个内置对象.
        用他动态获取虚拟路径
        ${pageContext.request.contextPath}
	cookie 是一个map集合.
        ${cookie.cookie的键名.value}
		${cookie.msg.value}

JSTL标签库

  c:if标签
        1. 属性：
            * test 必须属性，接受boolean表达式
                * 如果表达式为true，则显示if标签体内容，如果为false，则不显示标签体内容
                * 一般情况下，test属性值会结合el表达式一起使用

        2. 注意：c:if标签没有else情况，想要else情况，则可以在定义一个c:if标签
        //test属性中如果放的是el表达式. el表达式的大括号外边千万不要有空格
 foreach:相当于java代码的for语句
        1. 完成重复的操作
            for(int i = 0; i < 10; i ++){

            }
            * 属性：
                begin：开始值
                end：结束值
                var：临时变量
                step：步长
                varStatus:循环状态对象
                    index:容器中元素的索引，从0开始
                    count:循环次数，从1开始
        2. 遍历容器
            List<User> list;
            for(User user : list){

            }

            * 属性：
                items:容器对象 //items中的el表达式大括号前后也不要有空格
                var:容器中元素的临时变量
                varStatus:循环状态对象
                    index:容器中元素的索引，从0开始
                    count:循环次数，从1开始

<%@ taglib prefix="fn" uri="http://java.sun.com/jsp/jstl/functions" %>

   <c:if test="${fn:startsWith(name, '张')}">

       小张你来了
